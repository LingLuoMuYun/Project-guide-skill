# 后端项目分析

适用于后端服务、API 项目、微服务、数据处理服务等项目形态。

## 关键问题

- 使用什么语言和 Web 框架？
- 服务入口在哪里？
- 路由如何组织和注册？
- 中间件链是什么顺序？
- 请求/响应如何序列化？
- 数据库/缓存/消息队列如何连接？
- 错误如何统一处理和返回？
- 认证和授权方案是什么？
- 配置如何加载（环境变量、配置文件、配置中心）？
- 日志、监控、链路追踪如何集成？
- 有没有定时任务、后台 worker、消息消费者？

## 需要追踪的流程

```text
HTTP 请求
-> 网关/负载均衡
-> 中间件链（认证 -> 限流 -> 日志 -> 参数校验）
-> 路由匹配
-> Controller/Handler
-> Service/业务逻辑
-> Repository/DAO（数据访问）
-> 数据库/缓存/外部 API
-> 响应序列化
-> HTTP 响应
```

## 入口点识别

| 类型 | 常见位置 |
| --- | --- |
| HTTP 服务入口 | `main.*`, `app.*`, `server.*`, `index.*`, `cmd/` |
| 路由注册 | `router/**`, `routes.*`, `handler/**`, `controller/**` |
| 中间件 | `middleware/**`, `filter/**`, `interceptor/**` |
| 数据库连接 | `db.*`, `database.*`, `connection.*`, `datasource.*` |
| ORM 模型 | `models/**`, `entity/**`, `schema/**`, `migrations/**` |
| 服务层 | `services/**`, `usecase/**`, `domain/**` |
| 配置入口 | `config/**`, `.env`, `application*.yml`, `settings.*` |
| 定时任务 | `cron/**`, `jobs/**`, `tasks/**`, `scheduler/**` |
| 消息消费 | `consumer/**`, `listener/**`, `subscriber/**` |

## 架构分层

```
┌────────────────────────────────┐
│  Transport Layer (HTTP/gRPC)   │ ← 路由、中间件、序列化
├────────────────────────────────┤
│  Application/Service Layer     │ ← 用例编排、事务、权限
├────────────────────────────────┤
│  Domain Layer                  │ ← 核心业务逻辑
├────────────────────────────────┤
│  Infrastructure Layer          │ ← DB、缓存、MQ、外部 API
└────────────────────────────────┘
```

### 分层检查

- 是否存在跨层依赖（如 Controller 直接操作数据库）？
- Service 层是否有明确的边界和职责？
- 数据访问是否统一封装（Repository/DAO 模式）？

## 数据存储分析

### 关系型数据库
- 使用什么 ORM/驱动？
- Migration 文件在哪里？
- 连接池配置是什么？
- 有没有慢查询监控？

### 缓存
- Redis/Memcached 的使用场景
- Key 命名规范和过期策略
- 缓存穿透/击穿/雪崩的防护

### 消息队列
- Kafka/RabbitMQ/Pulsar 的使用场景
- Topic/Queue 命名
- 消费者的幂等性和重试策略
- 死信队列处理

## 外部依赖

| 依赖类型 | 检查项 |
| --- | --- |
| 数据库 | 连接串、连接池、读写分离、分库分表 |
| 缓存 | 地址、密码、DB 编号、集群模式 |
| 消息队列 | 地址、Topic、消费者组、分区 |
| 第三方 API | URL、超时、重试、熔断、降级 |
| 配置中心 | Apollo/Nacos/Consul 配置路径 |
| 注册中心 | 服务名、健康检查、负载策略 |
| 对象存储 | OSS/S3 Bucket、路径前缀 |

## 常见失败模式

- 数据库连接池耗尽
- 缓存雪崩导致数据库压力剧增
- 消息重复消费导致数据不一致
- 外部 API 超时拖垮整个请求链
- 配置热更新不生效
- 日志级别配置错误导致磁盘爆满
- 未处理 panic/rejection 导致进程退出
- 优雅关闭未实现导致请求中断
- 时区设置不一致导致数据错乱
- 大事务/长事务导致锁等待

## 推荐验证

- 启动服务并检查健康检查端点
- 调用核心 API 验证响应结构
- 查看日志输出是否正常
- 运行集成测试（如有）
- 检查数据库连接和 Migration 状态
- 检查消息队列消费是否正常
- 查看 Grafana/Prometheus 监控面板（如有）

## 安全审查要点

- 认证机制：JWT/Session/OAuth2/API Key
- 授权粒度：接口级/数据级/字段级
- 输入校验：参数类型/长度/范围/格式
- SQL 注入防护：参数化查询/ORM 防护
- 敏感数据：密码加密、日志脱敏、传输加密
- 限流和防护：速率限制、CC 攻击防护
- CORS 配置是否合理
- CSRF 防护是否到位
