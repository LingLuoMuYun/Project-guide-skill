# 项目指南技能

该技能用于快速接手陌生项目：把仓库整理成实用的项目学习指南，并生成一个交互式 HTML 页面供团队成员日常使用。重点回答：项目如何运行、应该从哪里开始读、主流程如何工作、目录结构怎么组织的、每个关键文件做什么、埋点在哪里、开发约定是什么、哪些地方容易出问题、还有哪些信息需要确认。

该技能适用于任何 AI Agent（Claude Code、Codex、Cursor、ChatGPT、Gemini 等）。在支持 skill 机制的平台上可以直接作为 skill 加载；在其他平台上可以将 `PROMPT.md` 的内容作为系统提示词复制使用。

## 操作原则

- 先检查再下结论。区分已观察事实、文档声明、运行验证、合理推断和待确认问题。
- 优先依据仓库真实结构，不套用泛化架构标签。
- 关键结论要绑定证据：文件路径、配置、命令输出、测试结果或文档。
- 不输出密钥、令牌、连接串等敏感值，只描述其用途和位置。
- 避免逐文件流水账。按职责、流程、边界、约定和风险解释。
- 未执行的命令要说明未执行及原因。
- 用户未指定语言时，面向中文项目默认输出简洁中文报告。
- **最终交付物**：在目标项目根目录创建 `project-guide/` 文件夹，生成一个自包含的交互式 HTML 指南。

## 分析深度

用户未指定时，默认使用 **standard**。

| 深度 | 范围 | 执行方式 | 输出重点 |
| --- | --- | --- | --- |
| quick | README、清单文件、顶层目录、脚本 | 只读发现 | 技术栈、关键文件、明显风险、下一步命令 |
| standard | quick + 入口文件、核心模块、配置、测试、项目结构树、埋点识别 | 安全只读 + 轻量验证 | 架构图、启动路径、核心流程、结构树、埋点清单、风险、阅读路线、HTML 指南 |
| deep | standard + 1 到 3 条核心流程和相关契约/测试 | 有针对性地运行测试、构建或类型检查 | 流程追踪、状态/契约、失败模式、覆盖缺口 |
| audit | 身份、权限、配置、状态、CI/CD、依赖等风险面 | 默认验证、dry-run、只读；更高风险动作需确认 | 证据清单、风险登记、修复优先级 |

## 核心流程

1. 仅在必要时澄清目标：入门、架构报告、开发指南、调试地图、重构规划、审计或交接。
2. 使用快速只读方式盘点仓库：
   - 顶层文件和目录
   - 文档、manifest、lockfile、配置、脚本、测试、CI、容器、环境模板
   - 需要跳过的生成物、vendor、缓存、构建目录
3. 识别技术栈：
   - 语言、框架、包管理器、运行时、构建工具
   - 基础设施和部署相关文件
4. 判断项目形态：
   - 前端、后端、全栈、库、CLI、monorepo、数据/机器学习、基础设施、移动/桌面、插件或混合系统
5. 找入口点：
   - 启动命令、应用根、路由/菜单定义、API client、服务入口、状态根、测试入口
6. 追踪主路径：
   - 输入/用户动作 -> 路由/分发 -> 组件/服务 -> 外部依赖 -> 状态/输出
7. **绘制项目结构树**：
   - 生成带注释的完整目录树（使用 ASCII 树形格式）
   - 按职责分组目录（核心代码、配置、测试、脚本、文档、构建产物）
   - 标注每个关键文件/目录的优先级（P0 必读 / P1 重要 / P2 参考 / P3 可跳过）
   - 标注特殊文件（★ 入口、● 核心模块、▲ 配置、⚠ 风险）
   - 绘制依赖方向图（自上而下的分层依赖）
   - 标记循环依赖、"上帝模块"、孤岛模块
8. **分析代码文件内容**：
   - 对每个 P0/P1 文件生成摘要卡片：职责、核心导出、内部依赖、被依赖方、关键逻辑、潜在问题
   - 按文件类型分类整理（入口、路由、API层、状态、组件、工具、类型）
   - 汇总模块间的调用关系
9. **识别埋点与追踪**：
   - 搜索埋点 SDK/库引用（gtag、mixpanel、sensors、sentry、aegis、自定义埋点等）
   - 搜索埋点调用模式（track(、logEvent、pageView、reportEvent 等）
   - 整理埋点事件清单（事件名、触发时机、所在文件、参数、验证状态）
   - 按类别分组：页面浏览、用户行为、业务流程、错误事件
   - 识别关键流程中缺失埋点的环节（覆盖缺口）
   - 检查埋点合规（敏感信息、用户退出机制）
10. 识别开发约定：
   - 目录布局、命名、服务/类型模式、状态使用、样式、错误处理、权限、测试
11. 评估验证情况：
   - 可用命令、实际执行命令、已证明内容、仍未验证内容
12. **构建快速导航索引**：
   - 按功能模块的跳转索引（模块名 → 关键文件列表）
   - 按文件类型的速查表
   - 常用命令速查
13. 生成 Markdown 指南：
   - 概览、技术栈、架构、结构树、关键目录、核心流程、模块摘要、埋点清单、开发路径、风险、开放问题、新人阅读路线、快速导航
14. **生成 HTML 项目指南**：
   - 在目标项目根目录创建 `project-guide/index.html`
   - 使用 `references/html-guide-template.md` 的模板结构
   - 填充分析数据到各章节
   - 包含：概览卡片、交互式目录树、Mermaid 架构图、模块详情、埋点表格、快速导航、阅读路线
   - 功能：左侧导航、全文搜索、暗色/亮色主题、响应式布局

## 项目结构树分析

### 目录树生成规范

1. **扫描目录**：从项目根目录开始，递归列出所有文件和目录（跳过 node_modules、.git、dist、build、vendor、__pycache__、.next 等）
2. **分组归类**：将目录按职责分组，使用中文注释标注
3. **优先级标注**：每个目录/文件标注 P0-P3 优先级
4. **特殊标记**：入口文件 ★、核心模块 ●、配置文件 ▲、风险文件 ⚠
5. **依赖方向**：分析 import 链，绘制分层依赖图

> 详细规范参见 `references/project-structure.md`。

### 代码文件内容分析

对每个关键文件生成结构化摘要，包括：
- 文件职责（一句话）
- 核心导出（函数/类/组件/类型列表）
- 内部依赖（引用了哪些项目模块）
- 被依赖方（被哪些模块引用）
- 关键逻辑（核心代码段说明）
- 潜在问题（技术债、性能、安全）
- 修改影响范围

## 埋点与追踪分析

### 搜索策略

1. 搜索常见埋点 SDK 的引用特征
2. 搜索自定义埋点函数的调用模式
3. 搜索埋点初始化和配置位置
4. 检查路由钩子中的自动埋点

### 输出清单

以表格形式整理所有埋点事件：
- 事件名、触发时机、所在文件（含行号）、参数列表
- 按类别分组：页面浏览 / 用户行为 / 业务流程 / 错误事件
- 标注验证状态（已验证 ✅ / 待确认 ⚠ / 缺失 ❌）
- 识别覆盖缺口并给出补充建议

> 详细搜索命令和识别模式参见 `references/tracking-analytics.md`。

## 快速导航

在 Markdown 报告和 HTML 指南中均需包含快速导航：

### 模块导航
按功能模块列出跳转链接（模块名 → 关键文件路径），让读者能快速定位到感兴趣的模块。

### 类型导航
按文件类型分组列出速查表（入口文件、API 服务、组件、工具函数、配置）。

### 命令导航
列出开发、构建、测试、部署中最常用的命令及说明。

## 生成 HTML 项目指南

### 生成位置

```
{目标项目根目录}/project-guide/index.html
```

### 生成要求

- 单文件、自包含（CSS/JS 内联，Mermaid 可选 CDN）
- 左侧固定导航栏 + 右侧内容区 + 顶部搜索栏
- 交互式目录树（可展开折叠）
- Mermaid 图表渲染架构和核心流程
- 埋点事件可搜索表格
- 暗色/亮色主题切换
- 响应式（桌面/平板/手机）
- 代码块一键复制
- 全文搜索（模块、文件、函数、埋点）
- 键盘快捷键（Ctrl+K 搜索、Ctrl+/ 侧边栏）

> 完整 HTML 模板和 CSS 规范参见 `references/html-guide-template.md`。

## 基线发现命令

优先使用这些只读命令：

```bash
rg --files
ls -la
cat README.md 2>/dev/null | head -80
```

PowerShell 等价命令：

```powershell
Get-ChildItem -Force
Get-ChildItem -Recurse -Depth 2 -File | Select-Object -First 120 FullName
Get-Content README.md -TotalCount 80
```

运行脚本前先检查脚本内容。除非用户明确要求或可以确认安全，不运行安装、升级、格式化、代码生成、迁移、种子数据、部署、apply、destroy 或会连接生产环境的命令。

## 证据标签

必要时使用以下标签：

- **已观察**：通过读取文件确认。
- **文档声明**：文档中写明，但未独立验证。
- **运行验证**：通过命令、测试或 API 输出确认。
- **推断**：由结构、命名或调用关系得出的合理结论。
- **待确认**：需要用户、后端、运维或运行环境确认。

## 默认输出

标准项目指南应包含：

1. 证据摘要
2. 项目概览
3. 技术栈
4. 目录和关键文件地图
5. **项目结构树（带注释的完整目录树 + 依赖方向图）**
6. 启动和运行路径
7. 架构和核心流程
8. 核心模块与职责
9. **代码文件摘要（P0/P1 文件的结构化分析卡片）**
10. 状态、契约和外部依赖
11. **埋点事件清单（按类别分组，含覆盖缺口）**
12. 开发工作流与约定
13. 测试和验证
14. 风险和开放问题
15. 推荐下一步
16. **快速导航（按模块 / 按文件类型 / 常用命令）**
17. 新人阅读路线
18. **HTML 项目指南（project-guide/index.html）**

如果是代码审查，先列风险。如果是调试任务，先列症状和隔离计划。如果是交接文档，先说明已可用内容、阻塞项，以及暂不建议修改的部分。

## 完成标准

收尾前应回答或标记待确认：

- 这是什么项目，提供什么价值？
- 如何安装、启动、构建、测试、部署或打包？
- 主要入口和运行边界在哪里？
- 最重要的数据流和控制流是什么？
- **项目结构树是否已生成（含优先级标注和依赖方向）？**
- **关键代码文件是否已生成摘要卡片？**
- **埋点事件是否已搜索并整理成清单？**
- 配置、状态、schema/契约和外部依赖在哪里？
- 哪些内容已经通过命令或运行行为验证？
- 最高风险区域和最稳妥的下一步是什么？
- 新人应该先读哪些文件？
- **快速导航索引是否已构建？**
- **HTML 项目指南（project-guide/index.html）是否已生成？**

## 跨平台使用

### 在支持 Skill 的平台上（Claude Code / Codex）

将本仓库克隆到 skill 目录，直接通过 `/project-guide-skill` 或 `$project-guide-skill` 调用。参考文件（`references/`）会被按需加载。

### 在其他 Agent 上（ChatGPT、Cursor、Gemini 等）

使用 `PROMPT.md`——这是一份自包含的通用 prompt，不依赖 `references/` 目录。将其中 `---` 分隔线之间的内容复制到任何 AI Agent 的对话中即可使用。
